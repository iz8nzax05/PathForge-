PATHFORGE PROJECT COMPREHENSIVE SUMMARY
Last Updated: 2025-01-27

OVERVIEW
========
PathForge is a comprehensive interactive story creation and visualization platform consisting of two main applications: PathForge 1.1 (story visualizer) and Nodepad 1.1 (node-based note-taking tool). The project demonstrates sophisticated software architecture with plugin-based modularity, professional user interfaces, and robust data management systems. The platform supports creating, editing, and visualizing interactive choice-based stories with a professional-grade node-based interface.

CORE APPLICATIONS
=================

Story Format:
Each story file follows a structured format:
```
N: [Node ID Number]
T: [Title of this story section]
S: [Main story text that the player reads]

A: [Choice text] -N1
B: [Choice text] ->N2
C: [Choice text] =N3
D: [Choice text] N4
E: [Choice text] -N5
F: [Choice text] ->N6
G: [Choice text] =N7
H: [Choice text] N8
```

Flexible Linking Formats Supported:
- `A: Take the path N4` (simplest - no connector required)
- `A: Take the path - N4`
- `A: Take the path -> N4`
- `A: Take the path = N4`
- `A: Take the path > N4`
- Any amount of whitespace allowed around connectors

EXTENSIBILITY
=============

Plugin Development:
The plugin architecture allows for easy extension of functionality without modifying core application code. New plugins can be developed by:
- Extending the base Plugin class
- Implementing required abstract methods (initialize, event handlers)
- Registering with the PluginManager
- Handling events appropriately
- Following consistent plugin patterns

Plugin Development Pattern:
```python
class MyPlugin(Plugin):
    def __init__(self):
        super().__init__("MyPlugin")
    
    def initialize(self, app):
        self.app = app
    
    def on_click(self, app, event):
        # Handle click event
        return True  # Consume event if handled
```
